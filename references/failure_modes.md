# 常见失败模式（用于反证与风控）

把这张表当成“强制检查清单”，在结论前逐条过一遍。

| 失败模式 | 公开信号 | 如何防范 |
|---|---|---|
| 把“热门受益者”误判成“必经 chokepoint” | 叙事里混淆 chokepoint 与 bottleneck；缺少“拿掉它会卡住吗”的反事实分析 | 强制画完供应链图，并写出“无它路径”的替代链条与成本/周期 |
| 叙事对，但节奏错 | 股价/成交量先爆发，经营数据尚未确认；公司表述“无重大进展” | 把“交易催化”和“经营验证”分开评分；仓位与催化剂时钟绑定 |
| 供给紧张来自政策，而非纯需求 | 出口许可、制裁、跨境审批导致的短缺/延迟 | 把地理与政策单列风险因子；对政策驱动短缺避免过度外推 |
| 产业路线切换过快 | 多路线并行（MSA/标准分裂）、替代方案快速成熟 | Bear case 必须包含“替代路线提前成熟”与时间窗口 |
| 股价先涨，财务后到（估值挤压） | 估值过热、波动率飙升、回撤风险显著 | 将“估值消化能力/流动性”纳入纪律：越热越小心 |
| 实体基础设施扩张慢于需求扩张 | 交期极长（>12m 或更久）、设备安装与认证周期被低估 | 对 power/grid/fab/substrate 等 thesis，强制写入交期+认证的时间模型 |
